# 鮎釣り記録アプリ

AndroidスマホのChromeで開き、ホーム画面に追加して使える鮎釣り記録用のPWAです。圏外でも記録追加・編集・削除・一覧表示ができるよう、アプリ本体はService Workerでキャッシュし、記録データはIndexedDBに保存します。

## 主な機能

- 1件の釣行記録を「共通情報」「午前」「午後」に分けて保存
- 河川、午前/午後のポイント名、道具を候補選択または直接入力
- 午前だけ、午後だけの入力でも保存
- 午前釣果、午後釣果、合計釣果の自動表示
- 午後欄へ「午前と同じ」で一括コピー
- 記録一覧、詳細編集、削除
- 入力項目の追加、表示名変更、表示/非表示、並び替え、必須/任意切り替え
- 選択肢の追加、編集、削除
- CSV出力
- JSONバックアップ出力、JSON復元
- LINEへ送る
- Android共有、共有文コピー

## 午前、午後を分けて記録する使い方

1. 「追加」を開きます。
2. 「共通情報」タブで、日付、河川、天気、気温、共通メモを入力します。漁協は初期非表示ですが設定で表示できます。
3. 「午前」タブで、開始時間、ポイント名、水温、水量、水濁り、川の状態、苔、使用竿、水中糸、ハナカン、針、釣果、最大サイズ、メモを入力します。
4. 「午後」タブで、午後の同じ項目を入力します。
5. 「記録を保存」を押します。

午前だけ、または午後だけでも保存できます。午前釣果数と午後釣果数から合計釣果数を自動計算します。

## 漁協と候補＋直接入力

漁協は共通情報の項目ですが、初期状態では非表示です。漁協、河川、ポイント名、使用竿、水中糸、ハナカン、針は、候補からタップして選ぶことも、候補にない値を直接入力することもできます。

入力欄に文字を入れると候補が絞り込まれます。候補をタップすると入力欄に反映されます。

候補にない値を入力して記録を保存すると、「この値を候補に追加しますか？」と確認されます。追加すると次回から候補に出ます。追加しなくても、その記録には入力した値が保存されます。

## 使用しかけの廃止

「使用しかけ」は今後の入力画面、CSV出力、LINE共有文には表示しません。過去の記録データに残っている値は削除せず、非表示扱いにしています。

## 水中糸、ハナカン、針

午前・午後それぞれで、使用竿の次に「水中糸」「ハナカン」「針」を入力できます。これらも候補から選択でき、候補にない値は直接入力できます。

初期候補は `options.json` で管理しています。

## 午後の「午前と同じ」ボタン

午後タブの上部に「午前と同じ」ボタンがあります。押すと、午前の水温、水量、水濁り、川の状態、苔、使用竿、水中糸、ハナカン、針、釣果、最大サイズ、メモを午後へコピーします。

午後にすでに入力済みの値がある場合は、上書き確認が出ます。コピー後も午後の内容は自由に修正できます。

## 入力項目や選択肢の変更方法

「設定」画面は3ページに分かれています。

1. 入力項目の設定
2. 選択肢の設定
3. バックアップ・初期化・アプリ設定

### 入力項目の設定

入力項目はカード形式で表示されます。「編集」を押した項目だけ詳細編集できます。

- 表示名を変更できます。
- 共通情報、午前、午後の区分を変更できます。
- 入力タイプを変更できます。
- 並び順を数字で変更できます。
- 表示/非表示を切り替えできます。
- 必須/任意を切り替えできます。
- 新しい入力項目を追加できます。

項目を使わなくなった場合は削除ではなく非表示にしてください。非表示にしても過去の記録データはすぐには消えないため、過去の記録を開いたときに当時のデータを守りやすくなります。

### 選択肢の設定

選択肢はカテゴリごとに折りたたみ表示されます。漁協、河川、ポイント名、天気、水量、水濁り、川の状態、苔、使用竿、水中糸、ハナカン、針などを管理できます。

追加・編集時は前後の空白を削除し、重複登録を防ぎます。

## CSVとJSONの保存・共有

バックアップ画面は「保存する」「共有する」「復元する」の3つのメニューに分かれています。

- 保存する: CSVやJSONバックアップをスマホに保存します。
- 共有する: CSVやJSONバックアップをLINE、Google Drive、メールなどへ送ります。
- 復元する: JSONバックアップから記録を戻します。

- CSVを保存: 記録を表計算や集計に使えるCSVファイルとして端末に保存します。
- CSVを共有: 対応端末ではAndroidの共有画面からLINE、Google Drive、メールなどへ送れます。
- JSONバックアップを保存: アプリへ復元できる完全バックアップをJSONファイルとして端末に保存します。
- JSONバックアップを共有: 対応端末ではJSONバックアップを共有できます。

CSVは閲覧・集計用、JSONは復元用です。LINE送信は一時共有向きなので、長期保管はGoogle Drive、パソコン、NASなどにも保存することをおすすめします。

JSONから復元すると、現在の記録や設定がバックアップ内容に置き換わる可能性があります。復元前に必要なデータを保存してください。

## TOPカレンダー

TOPページには、その月のカレンダーを表示します。記録がある日は日付の下に小さい「●」が表示されます。日付をタップすると、その日の記録がすべてカード形式で表示され、編集ボタンから記録を開けます。前月、翌月ボタンのほか、年と月を変更して別の年月も確認できます。

## 検索ページ

検索ページでは、開始日、終了日、月、河川、ポイント名、天気、気温範囲、水温範囲、水濁り、川の状態、苔で検索できます。

検索条件は「基本条件」と「詳細条件」に分かれています。基本条件には開始日、終了日、月、河川、ポイント名を表示します。天気、気温範囲、水温範囲、水濁り、川の状態、苔は「詳細条件を開く」から表示します。

- 釣行日数: 検索結果に含まれる日付の数です。同じ日に複数記録があっても1日として数えます。
- 合計釣果: 検索結果の午前釣果と午後釣果の合計です。
- 平均釣果: 合計釣果を釣行日数で割った値です。
- 気温、水温は下限と上限を入れて範囲指定できます。空欄の場合は条件なしです。

検索結果をさらに条件で絞り込むと、その対象データを使って使用竿、水中糸、針のベスト3を表示します。ベスト3は平均釣果順です。未入力の道具はランキング対象外、釣果数が未入力の記録は0匹として扱います。使用日数は、その道具を使った日付の数です。

## 編集画面の操作

編集画面では「変更を保存」と「その他の操作」が横並びで表示されます。共有、LINEへ送る、コピー、削除は「その他の操作」を押して下から開くメニューから使います。削除は確認メッセージが出ます。

## 写真機能 第1段階

共通情報に「写真」欄があります。写真を追加すると、長辺1280px程度・JPEG品質約76%の圧縮コピーとしてアプリ内のIndexedDBに保存します。元写真をそのまま保存するのではなく、記録には写真IDだけを持たせます。

- 写真は1件の釣行記録につき最大5枚までです。
- 「写真を追加」からスマホの写真を選択できます。複数枚選択できる端末ではまとめて追加できます。
- サムネイルをタップすると拡大表示できます。
- 写真の削除ボタンを押すと確認メッセージが出ます。
- 写真は通常のCSVやJSONバックアップには含めません。CSVには写真枚数だけを出力できます。
- 写真付きバックアップZIPや写真付き復元は、この第1段階では未実装です。

写真はスマホの通常の写真一覧ではなく、アプリ内データとして保存されます。既存写真を選んだ場合、元写真はスマホの写真アプリ側に残ります。確実に元写真を残したい場合は、先に通常のカメラアプリで撮影してから、このアプリで写真を選択する運用がおすすめです。

Chromeのサイトデータ削除、アプリ削除、スマホ初期化などでアプリ内写真が消える可能性があります。大事な写真は通常の写真アプリ、Google Drive、パソコン、NASなどにも保管してください。

## オフライン利用

このアプリはPWAとして動作し、一度オンラインで開いた後はオフラインでも起動できます。記録追加、編集、削除、一覧表示はIndexedDBに保存されたスマホ内データで動作します。ブラウザデータを削除すると記録が消える可能性があるため、釣行後はJSONバックアップを保存してください。

### バックアップ・初期化・アプリ設定

CSV出力、記録データのJSONバックアップ/復元、設定のJSONバックアップ/復元、初期設定から新しい項目だけ取り込む、初期設定に戻す、開発用リセットはこのページにまとめています。

初期化や削除に近い操作は確認メッセージが出ます。記録データを守るため、先にJSONバックアップを保存してください。

## fields.json と options.json

`fields.json` は入力項目の初期テンプレートです。項目の `id`、表示名、入力タイプ、選択肢キー、表示/非表示、並び順、必須/任意、共通情報・午前・午後の区分を管理します。

`options.json` は選択肢の初期テンプレートです。天気、川の名前、水量、水濁り、使用竿、使用しかけなどの初期選択肢を管理します。

スマホの設定画面で編集した内容はIndexedDBに保存されます。`fields.json` と `options.json` は初回登録用の初期テンプレートであり、アプリを起動するたびにユーザー設定を上書きするものではありません。

既存のIndexedDB設定がある場合、アプリはIndexedDB側を優先します。`fields.json` に新しい `id` の項目が追加された場合だけ、自動的に不足分を追加します。既存項目の表示名、表示/非表示、並び順、選択肢はユーザー設定を優先します。

## 新しい項目だけ取り込む方法

1. `fields.json` または `options.json` を更新します。
2. PWAキャッシュを更新するため、`sw.js` の `CACHE_NAME` を新しい値にします。
3. オンライン状態でアプリを開き直します。
4. 「設定」を開きます。
5. 「初期設定から新しい項目だけ取り込む」を押します。

既存のユーザー設定は上書きされず、同じ `id` がない新しい項目だけが追加されます。

## 初期設定に戻す方法

「設定」画面の「初期設定に戻す」を押すと、確認後に `fields.json` と `options.json` の内容で入力項目と選択肢を上書きします。

この操作では記録データ自体は削除しません。ただし、スマホで編集した項目名、並び順、表示/非表示、選択肢は初期状態に戻ります。

## 設定バックアップと復元

「設定」画面の「設定をJSONバックアップ」を押すと、入力項目と選択肢だけをJSONで保存できます。

「設定をJSON復元」から保存済みの設定JSONを読み込むと、入力項目と選択肢を復元できます。記録データとは別に設定だけを移したい場合に使います。

## fields.json / options.json を変更した後の反映方法

PWAではファイルがService Workerにキャッシュされます。`fields.json` や `options.json` を変更したのに反映されない場合は、`sw.js` の `CACHE_NAME` を変更してからオンライン状態でアプリを開き直してください。

それでも反映されない場合は、Chromeでアプリを閉じて再起動するか、開発中であればブラウザのサイトデータを削除して確認してください。実運用ではサイトデータを削除すると記録も消える可能性があるため、先にJSONバックアップを保存してください。

## 共有ボタン、コピー機能の使い方

記録詳細画面の「共有」は、対応端末ではAndroidの共有画面を開きます。対応していない場合は共有文をクリップボードへコピーします。

「コピー」を押すと、共有文だけをクリップボードへコピーできます。

## PWAとしての使い方

一度オンラインでアプリを開くと、HTML、CSS、JavaScript、manifest、アイコンがService Workerによりキャッシュされます。その後は圏外でもホーム画面のアイコンから起動できます。

記録データはIndexedDBに保存します。同じ端末・同じChromeであれば、ブラウザを閉じても記録は残ります。

ただし、Chromeのブラウザデータ削除、端末故障、別ブラウザへの変更では記録が消える可能性があります。定期的にJSONバックアップを保存してください。

## Androidスマホでホーム画面に追加する方法

1. AndroidスマホのChromeでアプリを開きます。
2. 右上のメニューを開きます。
3. 「ホーム画面に追加」を選びます。
4. 名前を確認して追加します。
5. ホーム画面に追加されたアイコンから起動します。

## オフライン動作確認方法

1. 一度オンラインでアプリを開く
2. AndroidスマホのChromeでホーム画面に追加する
3. スマホを機内モードにする
4. ホーム画面のアイコンからアプリを起動する
5. 新しい釣行記録を追加する
6. 記録を編集する
7. 記録一覧を表示する
8. ブラウザを閉じて再度開いても記録が残るか確認する

この手順で記録追加・編集・削除・一覧表示ができない場合は、オフライン対応として未完成です。

## CSV出力方法

1. 下部メニューの「保存」を開きます。
2. 「CSV出力」を押します。
3. `ayu-log-日付.csv` がダウンロードされます。

## JSONバックアップと復元方法

JSONバックアップ:

1. 下部メニューの「保存」を開きます。
2. 「JSONバックアップ出力」を押します。
3. `ayu-log-backup-日付.json` がダウンロードされます。

JSON復元:

1. 下部メニューの「保存」を開きます。
2. 「JSONから復元」を押します。
3. 保存していたJSONファイルを選びます。
4. 確認画面でOKを押すと、記録と設定が復元されます。
